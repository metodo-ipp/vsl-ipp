---
name: Orval-generated Zod schema imports
description: Correct import path for types/enums generated by Orval codegen from the OpenAPI spec
---

Types and enums generated by Orval codegen (e.g. `AreaSlug`, `QuizAnswerInput`) must be imported from the package root (`@workspace/api-client-react`), not from the internal generated file path (`@workspace/api-client-react/src/generated/api.schemas`). The internal path is not a public export and fails typecheck with `TS2307`.

**Why:** `lib/api-client-react/src/index.ts` re-exports everything from `./generated/api` and `./generated/api.schemas`, so all generated types are already available at the package root. A design/frontend subagent guessed the internal path and it silently passed at dev-server runtime (Vite doesn't type-check) but failed `pnpm typecheck`.

**How to apply:** When wiring frontend code to Orval-generated hooks/schemas, always import from the bare package name. Also remember `useGetArea`/similar parameterized query hooks require passing `queryKey: getGetAreaQueryKey(...)` inside `query` options when combined with `enabled` — omitting it fails typecheck (`queryKey` is required, not optional, in the generated options type).
